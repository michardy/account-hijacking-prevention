The purpose of this project is to provide website administrators with an open source piece of software to detect and prevent account hijacking.

About

I initially create this project to provide an open source solution to the problem of how to detect account hijacking. The initial project would use a system of modules to score the user's authenticity for a series of metrics. After writing the overarching system used by the modules I largely switched to focusing on one specific module, the keystroke dynamics module. While my work has largly focused on using keystroke dynamics, the use of modules leaves room for the inclusion of other forms of authentication.

I started out the keystroke dynamics module with the hope that I could achieve a 60 percent reliability. Fortunately I have been able to exceed this initial expectation and achieve a greater than 80 percent reliability. This still leaves considerable room for improvement.

The current keystroke dynamic system collects the average amount of time needed to change between different keys and compares it to previous averages. Currently the comparison is done by finding all the overlapping keys and finding the difference between the averages.

Design

My system has two parts a set of modules and server component.

Server

The server is the part of my project resposible for running module components, handling requests, and combining the results of the different modules. From a software perspective it handles all the things that are abstracted away so that modules do not have to do them.

When the server starts up it imports all of the modules and registers their functions and scripts. In ordinary use a protected page loads the scripts from the server and they collect data for processing by the server.
