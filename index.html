Account Hijacking Prevention

Repository  Issues

The purpose of this project is to provide website administrators with an open source peice of software to detect and prevent account hijacking.

About

I initially create this project to provide an open source solution to the problem of how to detect account hijacking. The initial project would use a system of modules to score the user's authenticity for a series of metrics. After writing the overarching system used by the modules I largely switched to focusing on one specific module, the keystroke dynamics module. The use of modules though still leaves room for expansion of the project past its current state.

I started out the keystroke dynamics module with the hope that I could achieve a 60 percent reliability. Fortunately I have been able to exceed this initial expectation and achieve a greater than 80 percent reliability. This still leaves considerable room for improvement.

The current keystroke dynamic system collects the average amount of time needed to change between different keys and compares it to previous averages. Currently the comparison is done by finding all the overlapping keys and finding the difference between the average.

Design

My system has two parts a set of modules that take different types of information about users and identifyies the user and server component that is responsible for gathering this information and running the modules.
